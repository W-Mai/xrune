# 咏唱语法

`ui! { … }` 宏接受的每一种形态。六轴：

- [上下文契约](context.md)：`:( … :)` 开篇头，每段咒文皆以此起手。
- [部件节点](widget.md)：语言的核心，具名或位置属性、括号可省、body 可省。
- [附魔](enchants.md)：`[expr, expr, …]` 方括号块，把任意数据挂到部件上。
- [控制流](control.md)：`if`、`walk … with …`、`@niche`、`match`。
- [`on` 事件咒符](on.md)：事件子句的两种形态（B 与 C）、带体或回调、限定或裸名、有参或无参。
- [被拒形态](errors.md)：parser 拒收的形态及理由。

DSL 在 **parse 期不带类型**。部件名是任意 ident，属性值是任意 `syn::Expr`。一切语义，何为「部件」、哪些 attr 算合法、某个事件名指什么，全在你绑的符文师里，不在 xrune 自身。
